# Feature-модули backend

Каждый модуль изолирован и содержит свой срез: контракты репозитория (интерфейс),
Prisma-реализацию, сервис (бизнес-логика), контроллер (HTTP) и/или бот-хендлеры.
Модули зависят от интерфейсов других модулей, а не от их внутренностей.

| Модуль          | Назначение                                                        | Этап |
|-----------------|-------------------------------------------------------------------|------|
| `users`         | Пользователи, регистрация по Telegram, таймзона, профиль          | 3    |
| `tasks`         | Задачи дня, статусы, парсинг плана (эвристика)                     | 3/4  |
| `reflection`    | Вечерние отчёты (DailyReport)                                      | 3/4  |
| `gamification`  | XP, уровни, streak, достижения (event-driven)                     | 6    |
| `pet`           | Виртуальный питомец, ленивый decay, реакции                       | 7    |
| `statistics`    | Инкрементальная агрегированная статистика                         | 3/6  |
| `notifications` | Учёт отправленных сообщений, идемпотентность рассылок             | 4    |
